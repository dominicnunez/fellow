# Feature Planning

Each feature folder contains the planning trail for one capability.

- `rfc.md` proposes the feature and records open scope.
- `adr.md` records the current technical decision and tradeoffs.
- `prd.md` describes product requirements and acceptance criteria.

Feature IDs are stable planning IDs, not release targets.

Auto-fix and watch mode are permission-gated and must not be implemented without explicit user approval.
